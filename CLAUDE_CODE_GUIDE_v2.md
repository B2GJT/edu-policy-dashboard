# 교육/HR 정책 트렌드 수집 시스템
## Claude Code 명령 가이드 v2

---

## 시스템 구성 요약

| 구성 요소 | 내용 |
|-----------|------|
| 수집 대상 | 정책브리핑(korea.kr) RSS, 매주 30건 |
| AI 분석 | Groq API — llama-3.3-70b-versatile (무료) |
| 대시보드 | GitHub Pages 자동 배포 |
| 알림 | Gmail SMTP 팀 이메일 발송 |
| 자동화 | Windows 작업 스케줄러 (`run_weekly.bat`) |

## 파일 구조

```
정부 정책 스크래핑/
├── policy_crawler.py       # 핵심 스크립트 (수집 + 분석 + 배포 + 이메일)
├── run_weekly.bat          # Windows 자동 실행 파일
├── dashboard.html          # 웹 대시보드 (자동 생성)
├── 가이드.md               # 운영 가이드 (API 키 교체 등)
├── CLAUDE_CODE_GUIDE_v2.md # 이 파일
└── history/                # 주차별 JSON 백업 (자동 생성)
    ├── 2026-W14.json       # 3월 30일 주차
    ├── 2026-W15.json       # 4월 6일 주차
    └── ...
```

```

```

```

```


## 대시보드 JSON 형식 참고

`history/` 폴더에 저장되는 JSON 구조:

```json
[
  {
    "title": "정책명",
    "source": "고용노동부",
    "date": "2026-04-06",
    "url": "https://www.moel.go.kr/...",
    "relevance": "높음",
    "keywords": ["AI훈련", "직업교육"],
    "business_opportunity": "사업 기회 설명 (2문장 이내)",
    "action": "즉시검토"
  }
]
```

`action` 값: `즉시검토` / `참고` / `모니터링`
`relevance` 값: `높음` / `중간`

## 문제 해결 FAQ

**Q. Groq API 오류가 발생해요**
- `console.groq.com` → API Keys에서 키 확인
- `policy_crawler.py` 상단 `GROQ_API_KEY` 값 교체

**Q. 이메일이 안 왔어요**
- Gmail 앱 비밀번호 만료 가능 → `가이드.md` 참고하여 재발급
- `run_log.txt` 열어서 오류 메시지 확인

**Q. GitHub Pages가 업데이트 안 됨**
- GitHub 토큰 만료 가능 → `가이드.md` 참고하여 재발급
- `run_log.txt` 열어서 오류 메시지 확인

**Q. 정책이 0건 수집됨**
- 정책브리핑 서버 일시 장애일 수 있음
- 잠시 후 다시 실행하거나 다음 날 시도
