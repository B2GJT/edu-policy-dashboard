# 교육/HR 정책 트렌드 수집 시스템 — 운영 가이드

> 마지막 업데이트: 2026-05-11 (팀 계정 B2GJT 이전 완료)

---

## 시스템 개요

매주 월요일 오전 10시에 자동으로 정부 정책을 수집하고, AI가 교육/HR 관점에서 분석한 뒤
웹 대시보드를 업데이트하고 팀원에게 이메일을 발송하는 시스템입니다.

```
매주 월요일 10:00 자동 실행
        ↓
연합뉴스·한겨레 RSS 5개 소스 수집 (200건+)
        ↓
키워드 1차 필터링 (40건으로 압축)
        ↓
Groq AI 분석 (교육/HR 관련 정책 필터링 + 부처별 분류)
        ↓
history/ 폴더에 JSON 저장
        ↓
dashboard.html 생성
        ↓
GitHub Pages 자동 업로드
        ↓
팀원 이메일 자동 발송
```

## 파일 구조

```
정부 정책 스크래핑/
├── policy_crawler.py   # 핵심 스크립트 (수집 + 분석 + 발송)
├── run_weekly.bat      # Windows 자동 실행 파일
├── dashboard.html      # 웹 대시보드 (자동 생성)
├── 가이드.md           # 이 파일
├── history/            # 주차별 분석 결과 (자동 생성)
│   ├── 2026-W14.json
│   └── 2026-W19.json  # 2026-05-11 최신
└── run_log.txt         # 실행 로그 (자동 생성)
```

## 주요 설정값 위치

민감 정보는 `.env` 파일에서 관리합니다. 메모장으로 열어서 수정하세요.

```
# .env 파일
GROQ_API_KEY=gsk_...        ← 재발급 시 여기만 교체
GITHUB_TOKEN=ghp_...        ← 재발급 시 여기만 교체
EMAIL_PASSWORD=xxxx xxxx    ← 재발급 시 여기만 교체
```

이메일 수신자 등 민감하지 않은 설정은 `policy_crawler.py` 상단에서 수정하세요.

```python
EMAIL_RECIPIENTS  = ["수신자1", "수신자2", ...]   ← 인원 추가/삭제
```

## 자동 실행 설정 (Windows 작업 스케줄러)

작업 스케줄러에 아래 이름으로 등록되어 있습니다:

- **작업 이름:** 정부정책스크래핑
- **실행 주기:** 매주 월요일 오전 10:00
- **실행 파일:** `run_weekly.bat`

등록 여부 확인 또는 재등록이 필요할 때 PowerShell에서 실행:

```
# 등록 확인
schtasks /query /tn "정부정책스크래핑" /fo LIST

# 재등록 (기존 덮어쓰기)
schtasks /create /tn "정부정책스크래핑" /tr "C:\Users\FC_임경민\Desktop\작업\정부 정책 스크래핑\run_weekly.bat" /sc WEEKLY /d MON /st 10:00 /f
```

## API 키 재발급 방법

### Gmail 앱 비밀번호
1. myaccount.google.com/apppasswords 접속
2. 기존 항목 삭제 → 새로 만들기
3. `policy_crawler.py` 상단 `EMAIL_PASSWORD` 교체

### Groq API 키
1. console.groq.com 접속 → API Keys
2. 기존 키 삭제 → Create API Key
3. `.env` 파일의 `GROQ_API_KEY` 교체

### GitHub 토큰
1. github.com → Settings → Developer settings → Personal access tokens
2. 기존 토큰 삭제 → Generate new token (classic)
3. **B2GJT 계정**으로 발급 (저장소: `B2GJT/edu-policy-dashboard`, `repo` 권한 체크)
4. `.env` 파일의 `GITHUB_TOKEN` 교체

## 문제 해결

**실행했는데 정책이 0건 수집됨**
- 연합뉴스 또는 한겨레 서버가 일시적으로 불안정한 경우입니다
- 잠시 후 다시 실행하면 보통 복구됩니다
- ※ 기존 정책브리핑(korea.kr)은 2026-04-27부로 봇 차단으로 사용 불가

**이메일이 안 왔어요**
- Gmail 앱 비밀번호가 만료됐을 수 있어요 → 재발급 후 교체
- `run_log.txt` 파일 열어서 오류 메시지 확인

**대시보드가 예전 내용으로 보임**
- GitHub Pages 캐싱 때문입니다. Ctrl+F5 로 강제 새로고침하세요
- GitHub 토큰이 만료됐을 수 있어요 → 재발급 후 교체
- `run_log.txt` 파일 열어서 오류 메시지 확인

**Claude Code에 문의할 때 이 문장 사용**
```
run_log.txt 읽어서 오류 원인 파악하고 고쳐줘.
```
